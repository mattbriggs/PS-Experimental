Use this token with any tool or application that makes API calls (such as cURL,WebDAV, Postman, or custom scripts). Create separate tokens for different applications or scripts.This enables you to revoke a token without affecting other integrations. Context When single sign-on (SSO) isconfigured in the CCMS, you have to generate a token. Otherwise, you can use your CCMS password. However, for security reasons, using a token is strongly recommended regardless of whether SSOis configured. Procedure 2 In a web browser, navigate to https://{organizationId}/tools/token-management/tokens.xql Where {organizationId} is the subdomain you use to accessHeretto CCMS. For example, if you accessHeretto CCMSat https://thunderbird.heretto.com, use thunderbird as the organizationId. Example: https://thunderbird.heretto.com/tools/token-management/tokens.xql 3 In theToken namefield, add a meaningfulname for the token. The name will be visible inthe tokens interface. Example: Oxygen Webdav Token, or CI/CD Pipeline, or Publishing Script 5 Copy the login and password (token) information and keep it in a secure place. . IMPORTANT:Once you navigate away from the page with your login and token, you won't be able to see the token again. Result Your token is now active. It will remain active until you manually invalidate it. File Access Documents Get Document Information You use the GET method to retrieve document information. Overview HTTP Method  GET  URI Webhooks use the internal XQuery Authentication Utils. No Authentication No authentication is an optionfor webhook requests. In this case, webhooksare sent to the provided URLs in the target system. No authentication headersare included in the webhookrequest to provide extra layers of security when an event is triggered. HTTP Header Authentication You can configure webhooks to include HTTP headers in the request. You can use this to configure Basic authentication orBearer authentication. For Basic authentication,you configure a username and password for the target system, and the Basicauthentication fields are included in the header of the HTTP webhook request. Bearer authentication is generally used for OAuth but can also be used forstored key authentication. OAuth 2.0 You can configure webhooks to fetch an access token using the client credentials OAuth method before sending the webhook request. This access token is then included as an HTTP header of the webhookrequest. To configure OAuth using client credentials authentication,you must provide the following: . Client ID . Client Secret . Token URL (this is a URL of an authorizationservice at the target service) Request Signature Header For additional security, to each webhook request header,Heretto CCMSadds an SHA-256 hash computed on thebasis of the request properties. To harden the webhooks security, you can define a secret that will be taken into account during the hashcomputation. .
